📦 Сборка

Файл — выгрузка из 1С (Excel / CSV / JSON) с колонками: Номенклатура, Штрихкод, Количество (необязательно: Артикул, Заказ, Клиент, Маркировка).

☁ Заказы на сервере

🔒 Заказ закрыт менеджером — сборка по нему завершена. Изменения больше не принимаются.

Завершение сборки

Отмечает заказ завершённым для менеджера. Если есть недостача — спросит подтверждение и комментарий.

Справочник товаров

Каталог ведёт менеджер в кабинете (office). Здесь — только просмотр, обновляется с сервера при запуске приложения. Нашли ошибку (например, штрихкод на товаре не совпадает) — откройте позицию в заказе и нажмите «⚠ Сообщить о проблеме»: менеджер увидит её на вкладке «Проблемы» и исправит.

Обмен с 1С по HTTP

Общий каталог на GitHub

Если указаны репозиторий и токен, приложение читает и сохраняет catalog.json и orders.json прямо в репозитории (работает и с приватным). Без токена — только чтение файлов, лежащих рядом с приложением (публичный репозиторий). Токен: github.com → Settings → Developer settings → Fine-grained tokens, доступ только к этому репозиторию, право «Contents: Read and write».

Сборщик

Когда два сборщика работают в одном заказе, позиция, которую сейчас собирает другой, подсвечивается «⏳ имя». Собранное обоими объединяется автоматически (нужны репозиторий и токен ниже).

Сканирование

Bluetooth-сканер (режим HID/клавиатура) работает всегда — просто сканируйте на экране заказа. Коды Честного знака (DataMatrix) распознаются и привязываются к товару по GTIN.
